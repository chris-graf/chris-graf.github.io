Positioning: Relative, Fixed, and Absolute

2015 July 19

display: static;

Yes, the title was a bit of a lie. Before we talk about relative, fixed, and absolute elements, we have to acknowledge
static
.
static
is the default for any otherwise undeclared elemnt, but it's also a bit of a contradiction. On the one hand it's the renegade that plays by its own rules, the James-Dean-in-Rebel-Without-a-Cause of stylesheets. On the one hand, it's the no-fun stoic whose actual rules are "don't go anywhere or do anything," like the James-Dean's-dad-in-Rebel-Without-a-Cause stylesheets. Basically,
static
elements go wherever the normal HTML flow tells them to, and nowhere else. You tell a static element to
top: 10px
, it pretends not to hear you and stays right where the document flow (i.e. HTML) puts it.

display: relative;

Now that we understand
static
,
relative
is easy.
relative
elements go with flow of the page, but they are also happy to directions from your stylesheets to shift up, right, left, or down from where they would normally be situated. If you know you need to move your element somewhere, but you're not sure how, try
relative
. Bam, done.

display: fixed;

Here's where it gets trickier.
fixed
positioning orients an element within the browser window (also known as the 'viewport'). An element with
fixed
positioning will always stay in the same spot relative to the window border regardless of scrolling or resizing. If you want to set navigation element on a long document, fixed is the way to go. While the ability to glue elements to the screen is quite valuable, remember that the more powerful the tool, the more trouble when it backfires. In our current media landscape where a consumer may go through a dozen different screen resolutions in a single day, responsive CSS through
@media
queries is often necessary to ensure your elements show up properly at all screen sizes.

For an example of
fixed
in action, check out the top and bottom bars on this very page.

display: absolute;

We saved the best for last.
absolute
is perhaps the most flexible positioning element, because it provides the most control over where elements appear on the page itself. When you give an
absolute
element styling directions, it places itself in relation not to the document or the browser window, but the containing element, as long as that container
relative
or
absolute
element. If there's no containing element, it behaves like
relative
. But if there is one, the container becomes the new point of reference. You might have
absolute
links in a
relative
navbar, or
absolute
pictures in a
relative
album grid. Like the other modes, mastering
absolute
positioning is essnetial toward crafting layered, expressive documents.
