Arrays and Hashes

2015 July 26

Arrays and hashes are both ordered collections of objects. Any Ruby object can be stored in an array or a hash, including other arrays and hashes. The essential difference between the two is how their conets are ordered and accessed, i.e. how you view and handle the objects you store inside them.


Arrays are organized through the simple means of a numeric index, with the first object stored at index [0]. The index refers only to the relative position of the object in an array, so reordering an array will change which index corresponds to which object.

types_of_pie = ["apple", "blueberry", "cherry"]
=> ["apple", "blueberry", "cherry"]
types_of_pie[0]
=> "apple"
types_of_pie.reverse!
types_of_pie[0]
=> "cherry"

Hashes are organized on "key-value pairs." Both the key and the value can be any Ruby object. A hash value is similar to an object in an array, except instead accessing a value by its current position in the array, you assign and use a key. While the index of an object in an array changes with reordering, the key-value pairing does not.

favorite_pies = {Danny: "apple", Eliza: "blueberry", Francis: "cherry"}
=> {:Danny=>"apple", :Eliza=>"blueberry", :Francis=>"cherry"}
(Note that this array assignment shorthand automatically converts the keys into symbols. To assign strings as keys without conversion, use the => operator.)
favorite_pies[:Danny]
=> "apple"
types_of_pie.reverse!
=> NoMethodError

Wait, what? That last step didn't work... because there's no reverse method for hashes. While hashes do remember the order in which their contents were assigned, they're not designed for easy re-ordering. You can sort hashes, but if you really care about the order of your elements, you might be better off with an array. In fact, here are a couple rules of thumb on deciding between the two:


When deciding whether to use an array or hash for a particular problem, the first question you should ask yourself is, "Do I care about the order of these objects within the collection?" If yes, go with an array (remember, you can make an array of arrays). Otherwise, ask yourself "Are the objects in my collection homogenous?" That is, are all the objects in your collection all examples of the same general thing?" (and thus, would not require special labels). If yes, stick with an array. Otherwise, go with a hash.
